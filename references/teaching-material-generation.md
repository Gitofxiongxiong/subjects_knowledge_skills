# 教学材料生成规范

当用户要求生成讲义、教案、练习题、试卷、课堂例题、课后作业等 Markdown 教学内容时，使用本规范。

本规范是线上 `教学内容生成` skill 的 API 版改写。公开 skill 不依赖服务器本地脚本，也不假设能直接读取文件系统。所有资料检索统一通过 `/api/kb`。

教学材料生成必须建立在知识库检索之上。使用本规范前，先按 `kb-api.md` 完成目录浏览、catalog 检索和页面读取；本文件只规定如何把已检索到的资料组织成可交付的教学内容。

## 核心原则

- 忠于知识库：知识库中有什么，就整理什么。不得凭空编造教材内容、题目、答案或教学细节。
- 先检索后生成：生成正文前必须完成 catalog 检索和页面读取。
- 不改编核心事实：尽量保留资料原有知识体系、概念顺序、公式和题目条件。
- 例题闭环：讲义和教案必须检索例题、答案和解析。若知识库确实无例题，按“缺失处理”执行。
- 图片可用：题目依赖图像、表格、实验装置、几何图、坐标图时，保留或提示 OSS 图片 URL。
- 质量门槛前置：结构、例题、公式、图片、表格、来源、内部痕迹都要在交付前检查。
- 不泄露密钥：最终内容和中间产物不得包含 API Key。

## 需求确认

生成前尽量确认以下信息：

- 学科：如 `高中物理`、`高中数学`。
- 教学阶段：新课、同步学习、一轮复习、二轮复习、三轮冲刺、考前复习。
- 资料类型：讲义、教案、练习、试卷、课堂例题、课后作业。
- 主题或知识点：如 `圆周运动`、`数列`、`化学平衡`。
- 教材版本、年级或章节：如人教版必修第二册、高一、高三。
- 输出长度和风格：基础巩固、考试冲刺、启发式、专题复习等。

缺失信息处理：

- 如果能从用户请求合理推断，就继续。
- 如果缺少学科或资料类型导致无法检索，先追问。
- 如果只缺少风格或长度，使用默认：结构完整、适合课堂交付、中等详略。

## 总流程

```text
确认需求
-> 规划小节
-> 为每个小节设计检索词
-> search/catalog 检索候选资料
-> read-pages 或 material-pack 读取正文与相邻页
-> 整理知识点、考点、例题、解析和训练
-> 生成 Markdown 教学材料
-> 质量检查
-> 输出终稿
```

如果用户已经给出小节，尊重用户小节。否则读取 `section-planning.md` 生成小节。

## 检索策略

### 目录定位

先按学科和阶段定位目录：

- 新课或同步：优先 `新课讲义`、`新课同步讲义`、`教材`、`新课教案`。
- 一轮复习：优先 `一轮复习` 或 `一轮复习讲义`。
- 二轮复习：优先 `二轮复习` 或 `二轮复习讲义`。
- 三轮或考前：优先 `三轮复习`、`考前最后一课`、`考前20天冲刺攻略`。
- 教材解释：优先 `教材` 或 `教材人教版`。

### catalog 检索

对每个小节至少准备：

- 核心概念检索词。
- 题型或例题检索词。
- 答案解析检索词。
- 若是复习材料，加入阶段或专题词。

示例：

```text
圆周运动
圆周运动 线速度 角速度
圆周运动 向心力 例题
圆周运动 临界问题 解析
```

### 读取页面

命中 `page_id` 后：

- 概念页：`before=1, after=1`。
- 例题页：`before=1, after=2`，防止题干和解析跨页。
- 多个命中页：用 `/material-pack` 合并，控制 `max_chars`。

读取后建立内部 source manifest：

```json
[
  {
    "section_id": "S1",
    "title": "圆周运动的基本概念",
    "sources": [
      {
        "page_id": "kb://...",
        "relative_path": "...",
        "relative_pdf": "...",
        "page_no": 1
      }
    ]
  }
]
```

正式讲义正文默认不展示 source manifest。用户要求溯源时，另列“资料来源”。

## 讲义结构

生成讲义时优先使用以下结构，可按主题调整：

```markdown
# 标题

## 学习目标

## 知识脉络

## 一、核心概念

## 二、公式、规律与适用条件

## 三、典型例题

### 例题 1

题干：

解析：

答案：

方法总结：

## 四、易错点与辨析

## 五、巩固训练

## 六、课堂小结

## 七、课后作业
```

要求：

- 学习目标要具体，可观察，可评价。
- 核心概念要按教学顺序组织，不堆砌。
- 公式必须说明物理量、单位或适用条件。
- 例题必须题干、解析、答案闭环。
- 巩固训练应和本讲知识点匹配。
- 不要出现“根据资料库第 x 页”这类内部痕迹，除非用户要求来源。

## 教案结构

生成教案时优先使用：

```markdown
# 标题

## 教学目标

## 教学重难点

## 学情分析

## 教学准备

## 教学过程

### 导入

### 新知探究

### 例题讲解

### 课堂练习

### 归纳总结

## 板书设计

## 作业设计

## 教学反思建议
```

要求：

- 教学过程要体现师生活动。
- 例题和练习必须与知识点对应。
- 若有实验、图像、表格或情境材料，应保留图片 URL 或说明查看图。

## 练习题与试卷结构

生成练习题、课后作业或试卷时：

- 按题型分组：选择题、填空题、解答题、实验题、综合题。
- 难度梯度：基础、提升、综合。
- 每题给出答案和解析，除非用户明确要求不带解析。
- 题目条件不得缺失，图像题要保留图片 URL。
- 不得把无答案的 OCR 片段伪造成完整题目。

## 缺失处理

完成规定检索后仍无可用资料时：

1. 明确缩小或调整检索词再搜一次。
2. 尝试同义词或教材常用表达。
3. 尝试相邻阶段资料，例如新课缺失时查教材，一轮缺失时查二轮专题。
4. 仍缺失时，在内部记录“知识库缺失”。
5. 最终输出中可说明“知识库未检索到足够材料，以下为基于通用学科知识的补充”，但不要把补充内容伪装成知识库内容。

## 质量检查

交付前逐项检查：

- 需求匹配：学科、阶段、资料类型、主题是否正确。
- 小节完整：用户明确要求的知识点是否全部覆盖。
- 来源充足：每个核心小节是否读取了相关资料。
- 例题闭环：题干、答案、解析是否齐全。
- 公式可读：公式、单位、符号解释是否清楚。
- 图片可用：图片 URL 是否保留，图像题是否提示看图。
- OCR 噪声：明显错字、断行、乱码是否整理过。
- 内部痕迹：终稿是否去掉内部绝对路径、调试记录、source manifest。
- 安全：终稿不含 API Key、服务器路径、环境变量值。

## 最终输出

默认输出可直接交付的 Markdown 教学材料。除非用户要求，不要附上冗长的检索过程。

如果用户要求来源，附独立小节：

```markdown
## 资料来源

- `relative_pdf`，page_no=...
- `page_id`
```

不要在学生版正文中混入内部调试信息。

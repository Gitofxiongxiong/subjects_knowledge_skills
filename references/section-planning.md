# 小节规划规范

在生成讲义、教案、练习题、试卷、PPT 内容或其他教学材料前，如果用户没有给出明确小节，先做小节规划。

小节规划只确定“应该拆成哪些小节、顺序是什么、每节检索什么资料”。不要在这个阶段生成正文。

## 输入识别

从用户请求中提取：

- 学科：例如 `高中物理`、`高中数学`。
- 教学阶段：新课、同步学习、一轮复习、二轮复习、三轮冲刺、考前复习等。
- 资料类型：讲义、教案、练习题、试卷、课堂例题、课后作业等。
- 主题范围：完整大专题、标准课时主题、窄知识点、题型方法。
- 显式知识点：用户明确写出的概念、公式、方法、题型、教材章节。
- 教材版本或年级：如人教版必修第二册、高一、高三等。

如果学科、教学阶段、资料类型缺失且无法从上下文合理推断，先简短追问。不要为了继续生成而随意假设。

## 主题颗粒度判断

### 具体知识点

例：`等差数列通项公式`、`抛物线定义`、`楞次定律`。

处理方式：小节少而聚焦，通常 3 到 5 节即可。只做必要前置补充，不扩展到整个大章。

### 标准课时主题

例：`圆周运动`、`电场强度`、`离子反应`、`基因分离定律`。

处理方式：按教师讲解顺序拆成 5 到 8 节，覆盖概念、规律、方法、题型、应用。

### 较宽泛上位主题

例：`圆锥曲线`、`牛顿运动定律`、`化学反应速率与平衡`。

处理方式：先限制边界，再拆成 6 到 10 节。不要无限扩展到整本书。

## 顺序原则

小节顺序以教师常用讲解顺序为主：

```text
前置情境或问题
-> 核心概念
-> 公式/性质/规律
-> 基本方法
-> 典型题型
-> 易错点
-> 综合应用
-> 小结与训练
```

如果用户请求本身有顺序，优先尊重用户表达。

## 小节规划输出格式

生成教学材料前，先内部得到如下结构。可以用 Markdown 表格或 JSON，但必须包含这些字段：

```json
[
  {
    "section_id": "S1",
    "title": "小节标题",
    "purpose": "本节教学目的",
    "knowledge_points": ["知识点1", "知识点2"],
    "retrieval_queries": ["用于 search/catalog 的检索词"],
    "expected_material": "概念讲解 / 例题 / 练习 / 答案解析 / 教案活动"
  }
]
```

`retrieval_queries` 要适合 `/search/catalog`：

- 使用教材常用短词。
- 同时准备宽词和窄词。
- 对题目需求加入 `例题`、`典例`、`练习`、`解析`、`答案`。
- 对复习需求加入 `一轮`、`二轮`、`专题`、`高考`、`易错` 等 stage 或关键词。

## 不要做的事

- 不要在小节规划阶段生成完整讲义正文。
- 不要为了补全而扩大主题边界。
- 不要脱离用户 query 擅自加入跨章节内容。
- 不要直接扫描服务器文件系统。
- 不要依赖旧的本地数据库脚本或 `query_tree.py`。
- 不要把内部规划过程原样塞进最终教学材料，除非用户要求展示规划。
